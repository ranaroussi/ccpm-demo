# Building Content-Rich Sites with MDX

MDX is a powerful format that combines Markdown with JSX, allowing you to create rich, interactive content. This post explores how to leverage MDX for documentation and blogging.

## What is MDX?

MDX lets you write JSX embedded inside Markdown. This means you can:

- Use React components in your content
- Create interactive elements
- Maintain readable, structured content
- Leverage the ecosystem of Markdown plugins

## Basic MDX Syntax

Here's how you can mix Markdown and React:

```mdx
# My Post Title

This is regular **Markdown** content.

<CustomComponent prop="value">
  And this is a React component!
</CustomComponent>

Back to Markdown again.
```

## Creating Custom Components

You can create reusable components for your content:

```jsx
// components/mdx/CodeBlock.jsx
export function CodeBlock({ children, language, title }) {
  return (
    <div className="code-block">
      {title && <div className="code-title">{title}</div>}
      <pre className={`language-${language}`}>
        <code>{children}</code>
      </pre>
    </div>
  );
}
```

Then use it in your MDX:

```mdx
<CodeBlock language="javascript" title="Example Function">
{`
function greet(name) {
  return \`Hello, \${name}!\`;
}
`}
</CodeBlock>
```

## Advanced Features

### Interactive Elements

Create engaging content with interactive components:

```jsx
// components/mdx/Counter.jsx
'use client';
import { useState } from 'react';

export function Counter({ initial = 0 }) {
  const [count, setCount] = useState(initial);

  return (
    <div className="counter">
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}
```

Use in MDX:

```mdx
Try this interactive counter:

<Counter initial={5} />
```

### Callout Components

Create attention-grabbing callouts:

```jsx
// components/mdx/Callout.jsx
export function Callout({ type = 'info', children }) {
  const styles = {
    info: 'border-blue-200 bg-blue-50 text-blue-800',
    warning: 'border-yellow-200 bg-yellow-50 text-yellow-800',
    error: 'border-red-200 bg-red-50 text-red-800',
  };

  return (
    <div className={`border-l-4 p-4 ${styles[type]}`}>
      {children}
    </div>
  );
}
```

Usage:

```mdx
<Callout type="warning">
  **Important**: Always validate user input before processing.
</Callout>

<Callout type="info">
  This is a helpful tip for your readers.
</Callout>
```

## Configuration

### Next.js Setup

Configure Next.js for MDX support:

```javascript
// next.config.mjs
import createMDX from '@next/mdx';

const withMDX = createMDX({
  options: {
    remarkPlugins: [],
    rehypePlugins: [],
  },
});

export default withMDX({
  pageExtensions: ['js', 'jsx', 'mdx', 'ts', 'tsx'],
});
```

### Plugin Ecosystem

Enhance MDX with plugins:

```javascript
import remarkGfm from 'remark-gfm';
import remarkToc from 'remark-toc';
import rehypeSlug from 'rehype-slug';
import rehypePrettyCode from 'rehype-pretty-code';

const withMDX = createMDX({
  options: {
    remarkPlugins: [remarkGfm, remarkToc],
    rehypePlugins: [
      rehypeSlug,
      [rehypePrettyCode, { theme: 'github-dark' }],
    ],
  },
});
```

## Best Practices

### Component Guidelines

1. **Keep components simple and focused**
2. **Make them reusable across content**
3. **Provide clear prop interfaces**
4. **Handle edge cases gracefully**

```jsx
// Good: Simple, focused component
export function ImageWithCaption({ src, alt, caption }) {
  return (
    <figure>
      <img src={src} alt={alt} />
      {caption && <figcaption>{caption}</figcaption>}
    </figure>
  );
}
```

### Performance Considerations

- Use dynamic imports for heavy components
- Optimize images with Next.js Image component
- Minimize client-side JavaScript in content

```jsx
// Dynamic import for heavy components
import dynamic from 'next/dynamic';

const HeavyChart = dynamic(() => import('./HeavyChart'), {
  ssr: false,
  loading: () => <p>Loading chart...</p>,
});
```
